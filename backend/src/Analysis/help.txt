# 📊 **Database Queries Module**

## 📋 **Общее описание**

Модуль `queries.py` предоставляет удобный Python-интерфейс для извлечения и анализа данных из базы данных авиаполетов AeroGuys. Он абстрагирует сложные SQL-запросы и предоставляет простые методы для получения информации о полетах, текущих позициях самолетов, статистике по аэропортам и многом другом.

---

## 🔧 **Требования и зависимости**

- Python 3.8+
- PostgreSQL (через Docker)
- Модуль `DB.dbManager` для подключения к базе данных
- Установленные зависимости из `requirements.txt`

---

## 📦 **Структура модуля**

Модуль содержит три основных класса данных и один основной класс для запросов:

### **Классы данных (Data Classes)**

| Класс | Описание | Назначение |
|-------|----------|------------|
| `FlightInfo` | Информация о полете | Содержит данные о конкретном рейсе: время, маршрут, длительность |
| `AircraftPosition` | Текущая позиция самолета | Координаты, высота, скорость, статус |
| `AirportStats` | Статистика аэропорта | Количество рейсов, вылетов/прилетов за период |

### **Основной класс**

| Класс | Описание |
|-------|----------|
| `DatabaseQueries` | Главный класс, содержащий все методы для запросов к БД |

## 🎯 **Функциональные возможности**

### **1. Запросы информации о полетах**

#### `get_flights_by_airport(airport, start_time, end_time, flight_type)`
Получение всех полетов через указанный аэропорт.

**Параметры:**
- `airport` - ICAO код аэропорта (например, 'UUEE' для Шереметьево)
- `start_time`, `end_time` - временной диапазон (datetime)
- `flight_type` - тип полетов: 'departure' (вылеты), 'arrival' (прилеты), 'both' (все)

**Возвращает:** список объектов `FlightInfo`

---

#### `get_flights_by_route(departure, arrival, start_time, end_time)`
Получение всех полетов по конкретному маршруту.

**Параметры:**
- `departure` - аэропорт вылета (ICAO код)
- `arrival` - аэропорт прилета (ICAO код)
- `start_time`, `end_time` - временной диапазон

**Возвращает:** список объектов `FlightInfo`

---

#### `get_flights_by_aircraft(icao24, start_time, end_time, limit)`
Получение истории полетов конкретного самолета.

**Параметры:**
- `icao24` - уникальный 24-битный код самолета
- `start_time`, `end_time` - временной диапазон
- `limit` - максимальное количество записей

**Возвращает:** список объектов `FlightInfo`

---

### **2. Текущие позиции самолетов**

#### `get_current_positions(min_lat, max_lat, min_lon, max_lon, min_altitude, on_ground_only, airborne_only, limit)`
Получение текущих позиций самолетов из последнего снапшота.

**Параметры фильтрации:**
- `min_lat`, `max_lat` - диапазон широты
- `min_lon`, `max_lon` - диапазон долготы
- `min_altitude` - минимальная высота (метры)
- `on_ground_only` - только на земле
- `airborne_only` - только в воздухе
- `limit` - максимум записей

**Возвращает:** список объектов `AircraftPosition`

---

#### `get_aircraft_by_callsign(callsign)`
Поиск самолета по позывному.

**Параметры:**
- `callsign` - позывной рейса (например, 'AFL123')

**Возвращает:** один объект `AircraftPosition` или `None`

---

### **3. Статистические функции**

#### `get_airport_stats(days)`
Статистика по аэропортам за последние N дней.

**Параметры:**
- `days` - количество дней для анализа

**Возвращает:** список объектов `AirportStats`, отсортированный по активности

**Пример вывода:**
```
Аэропорт: UUEE, Вылеты: 45, Прилеты: 52, Всего: 97 рейсов
```

---

#### `get_traffic_hourly(airport, days)`
Почасовое распределение трафика.

**Параметры:**
- `airport` - (опционально) код конкретного аэропорта
- `days` - период анализа в днях

**Возвращает:** словарь `{час: количество_рейсов}`

**Пример:**
```python
{
    0: 12,   # 00:00 - 01:00 - 12 рейсов
    1: 8,    # 01:00 - 02:00 - 8 рейсов
    ...
    23: 15   # 23:00 - 00:00 - 15 рейсов
}
```

---

#### `get_top_airlines(limit)`
Топ авиакомпаний по количеству рейсов.

**Параметры:**
- `limit` - количество компаний в топе

**Возвращает:** список кортежей `(код_авиакомпании, количество_рейсов)`

**Пример:** `[('AFL', 150), ('SBI', 120), ('RYR', 100)]`

---

### **4. Треки полетов**

#### `get_flight_track(flight_id)`
Получение подробного маршрута полета.

**Параметры:**
- `flight_id` - ID полета в базе данных

**Возвращает:** список точек маршрута, каждая точка содержит:
- `time` - время (datetime)
- `lat`, `lon` - координаты
- `altitude` - высота (метры)
- `track` - курс (градусы)
- `on_ground` - на земле (bool)

---

### **5. Экспорт данных**

#### `export_flights_to_csv(start_time, end_time, filename)`
Экспорт данных о полетах в CSV файл.

**Параметры:**
- `start_time`, `end_time` - временной диапазон
- `filename` - имя выходного файла

**Формат CSV:**
```
ICAO24, Callsign, Departure Time, Arrival Time, From, To, Duration (min)
abc123, AFL123, 2024-01-15 10:00, 2024-01-15 12:30, UUEE, LED, 150
```

### **Анализ загруженности аэропорта**
```python
# Статистика за 7 дней
stats = queries.get_airport_stats(days=7)

# Поиск конкретного аэропорта
sheremetyevo = next((s for s in stats if s.airport == 'UUEE'), None)
if sheremetyevo:
    print(f"Шереметьево: {sheremetyevo.total_flights} рейсов")

# Почасовое распределение
hourly = queries.get_traffic_hourly(airport='UUEE', days=7)
for hour in range(24):
    if hourly.get(hour, 0) > 0:
        print(f"{hour:02d}:00 - {hourly[hour]} рейсов")
```

### **Отслеживание конкретного рейса**
```python
# Поиск рейса по позывному
flight = queries.get_aircraft_by_callsign('AFL123')
if flight:
    print(f"Рейс {flight.callsign}:")
    print(f"  Координаты: {flight.latitude}, {flight.longitude}")
    print(f"  Высота: {flight.altitude} м")
    print(f"  Скорость: {flight.speed} км/ч")
    print(f"  Статус: {'На земле' if flight.on_ground else 'В воздухе'}")
```

## ⚠️ **Важные замечания**

### **Особенности работы**
1. **Временные метки** - все времена в UTC
2. **Пустые результаты** - нормальное явление, если данных еще мало
3. **Производительность** - рекомендуется использовать параметр `limit` для больших запросов

### **Возможные ошибки и их решение**

| Ситуация | Причина | Решение |
|----------|---------|---------|
| Нет данных в результатах | База данных пуста | Запустить `quick_poll.py` несколько раз |
| Медленные запросы | Слишком большой объем данных | Добавить фильтры или уменьшить `limit` |
| Ошибки подключения | Docker не запущен | Проверить `docker-compose ps` |
