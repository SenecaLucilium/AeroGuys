# Руководство пользователя — AeroGuys

## 1. Введение

**AeroGuys** — аналитическая платформа мониторинга авиационного трафика в реальном времени. Система собирает данные о рейсах из OpenSky Network, хранит их в базе данных PostgreSQL и предоставляет визуализацию через веб-интерфейс.

### Возможности системы:
- Просмотр текущих позиций воздушных судов на интерактивной карте
- Аналитика загруженности аэропортов и городов
- Статистика маршрутов и авиакомпаний
- Анализ параметров полётов (высота, скорость, фазы)
- Экспорт данных в форматах CSV и ZIP

---

## 2. Установка и запуск

### 2.1 Требования

- [Docker](https://docs.docker.com/get-docker/) и Docker Compose **или** Python 3.11+, Node.js 18+
- Учётные данные [OpenSky Network API](https://opensky-network.org/) (Client ID и Client Secret)

### 2.2 Запуск через Docker (рекомендуется)

1. Клонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd AeroGuys
   ```

2. Создайте файл `.env` на основе шаблона `.env.example` и заполните учётные данные:
   ```
   OPENSKY_CLIENT_ID=ваш_client_id
   OPENSKY_CLIENT_SECRET=ваш_client_secret
   ```

3. Запустите все сервисы:
   ```bash
   docker compose up --build
   ```

4. Откройте браузер:
   - Веб-интерфейс: [http://localhost:3000](http://localhost:3000)
   - API-документация: [http://localhost:8000/api/docs](http://localhost:8000/api/docs)

### 2.3 Запуск без Docker (разработка)

**Бэкенд:**
```bash
cd AeroGuys
python -m venv venv
source venv/bin/activate          # Linux/macOS
# или: venv\Scripts\activate      # Windows
pip install -r requirements.txt

cd backend/src
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

**Фронтенд:**
```bash
cd AeroGuys/frontend
npm install
npm run dev
```

> Убедитесь, что PostgreSQL запущен и переменные окружения `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` настроены.

## 3. Первый запуск

### 3.1 Загрузка данных

После запуска база данных пуста. Для начала работы необходимо загрузить данные.

**Способ 1 — Загрузить данные из OpenSky Network:**

1. В левой боковой панели перейдите в раздел **«Инициализация»** (иконка настройки).
2. Нажмите кнопку **«Загрузить из OpenSky»**.
3. Дождитесь завершения загрузки — система получит текущие позиции самолётов и данные о рейсах за последние 2 часа.

**Способ 2 — Загрузить рейсы из CSV:**

1. В разделе **«Инициализация»** выберите вкладку **«Загрузить CSV»**.
2. Нажмите **«Выбрать файл»** и укажите CSV-файл с данными рейсов.
3. Нажмите **«Загрузить»**.

### 3.2 Проверка статуса

На странице инициализации отображаются счётчики загруженных рейсов и состояний воздушных судов. Если данные есть, можно приступать к работе.

## 4. Разделы интерфейса

### 4.1 Реальное время

Страница отображает актуальные данные из последнего снапшота.

**Верхние карточки (KPI):**
- Всего воздушных судов в снапшоте
- Количество бортов в воздухе
- Количество бортов на земле
- Количество рейсов, активных в последние 24 часа

**Графики:**
- **Загруженность городов** — горизонтальная столбчатая диаграмма топ городов по общему числу рейсов
- **Фазы полёта** — круговая диаграмма: набор высоты, снижение, горизонтальный полёт
- **Скоростное распределение** — гистограмма скоростей воздушных судов
- **Страны воздушных судов** — круговая диаграмма по странам регистрации

**Таблицы:**
- **Самые быстрые самолёты** — топ по скорости в км/ч
- **Самые высокие самолёты** — топ по барометрической высоте в метрах

---

### 4.2 Аэропорты

Страница предоставляет аналитику аэропортов.

**Доступные виджеты:**
- Рейтинг аэропортов по загруженности (вылеты + прилёты)
- Ежедневная динамика конкретного аэропорта (вылеты и прилёты по дням)
- Пиковые часы — почасовое распределение рейсов
- Топ направлений из аэропорта (куда и как часто)
- Сравнительная пропускная способность нескольких аэропортов

**Как использовать:**
1. В поле поиска введите ICAO-код аэропорта (например, `EDDF` — Франкфурт, `LFPG` — Шарль-де-Голль, `UUEE` — Шереметьево).
2. Укажите период анализа (в днях).
3. Нажмите кнопку для получения данных.

---

### 4.3 Воздушные суда

Страница содержит детальную аналитику воздушных судов.

**Доступные функции:**
- **Текущие позиции** — таблица всех бортов из последнего снапшота с возможностью фильтрации по:
  - Координатному прямоугольнику (bbox)
  - Минимальной высоте
  - На земле / в воздухе
- **Поиск по позывному** — введите позывной (например, `AFL123`) для поиска конкретного борта
- **Типы ВС** — распределение по категориям ADS-B
- **Высотный профиль** — средняя высота и скорость по типам ВС
- **Неопознанные борты** — военные и борты без позывного
- **Бизнес-авиация** — лёгкие суда с нестандартными позывными

---

### 4.4 Маршруты

Страница предоставляет аналитику маршрутов.

**Доступные виджеты:**
- **Популярные маршруты** — топ пар аэропортов по частоте рейсов
- **Авиакомпании** — топ авиакомпаний по числу рейсов (по ICAO-коду из позывного)
- **Длительность рейсов** — гистограмма распределения продолжительности полётов (0–720 мин)
- **Эффективность маршрутов** — сравнение фактического времени с теоретической ортодромией

Для всех виджетов можно задать период анализа в днях (1–90).

---

### 4.5 Карта

Интерактивная карта всех воздушных судов из последнего снапшота.

**Функции карты:**
- Цветовая индикация:
  - Синие маркеры — самолёты в воздухе
  - Оранжевые маркеры — самолёты со скоростью > 700 км/ч
  - Серые маркеры — самолёты на земле
- Клик по маркеру — всплывающее окно с информацией: позывной, страна, скорость, высота, статус
- Фильтр по режиму: **Все / В воздухе / На земле**
- Автообновление каждые **30 секунд**

**Управление картой:**
- Прокрутка колесом мыши — масштаб
- Перетаскивание — перемещение
- Кнопка обновления — немедленная загрузка актуальных данных

## 5. Экспорт данных

В боковой панели доступны кнопки экспорта:

| Кнопка                | Формат | Описание                                      |
|-----------------------|--------|-----------------------------------------------|
| **Экспорт сырых данных** | CSV    | Все рейсы с полным набором столбцов           |
| **Экспорт аналитики**    | ZIP    | Несколько CSV-файлов с аналитическими данными |

Для экспорта рейсов за конкретный период используйте API: `GET /api/export/flights?start=...&end=...`

## 7. Часто задаваемые вопросы

**В. Данные не отображаются — что делать?**  
О. Перейдите в раздел «Инициализация» и загрузите данные из OpenSky. Если статус показывает 0 рейсов и 0 состояний — данных в базе нет.

**В. Ошибка «OpenSky API credentials not configured»**  
О. Убедитесь, что в файле `.env` заполнены переменные `OPENSKY_CLIENT_ID` и `OPENSKY_CLIENT_SECRET`. Перезапустите контейнер после изменения файла.

**В. Карта пустая**  
О. Карта отображает данные из последнего снапшота. Если снапшоты не загружены — выполните инициализацию данных.

**В. Аналитика по аэропорту показывает пустые данные**  
О. Убедитесь, что правильно введён ICAO-код (4 символа, например `EDDF`). Если рейсов через этот аэропорт в базе нет — данных не будет.

**В. Как остановить автообновление карты?**  
О. Автообновление карты происходит каждые 30 секунд. На текущей версии отключить его из интерфейса нельзя — это делается через исходный код.
